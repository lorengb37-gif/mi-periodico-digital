Hearst Magazines lanza Aura IQ: AI agents para advertising que duplicaron CTRs en primeras pruebas — creación de audiencias custom en tiempo real — RFPs respondidos en minutos en lugar de horas — debuta públicamente en Cannes Lions el lunes 22 · el primer media owner con infraestructura de ad ops AI end-to-end · lo que esto significa para el flujo de trabajo de planners y buyers 🧠

Hearst Magazines lanzó internamente Aura IQ —y lo presentará al mercado en Cannes Lions el lunes 22 de junio—, una plataforma que coloca AI agents sobre su producto de audience targeting Aura (lanzado en 2024, ya usado en 1 de cada 3 campañas de Hearst). Los AI agents de Aura IQ automatizan tres flujos de trabajo centrales del ad ops: creación de audiencias custom (antes horas de análisis manual, ahora minutos), análisis de performance de campaña en tiempo real, y respuesta a RFPs. En pruebas internas con decenas de RFPs en verticales como auto, retail, CPG, moda y beauty, Aura IQ documentó un 2x lift en clickthrough rate. El fundamento técnico: combina datos first-party de Hearst con señales contextuales y de comportamiento para generar recomendaciones de audiencia y campaña en tiempo real. La señal clave no es el producto — es el modelo: el media owner que automatiza su propio ad ops con IA cambia los términos de la negociación con compradores y agencias.

OpenAI ChatGPT Ads completa su suite de advertising: pixel de conversión + Conversions API + CPC bidding + self-serve ya disponibles — conversion-optimized campaigns rolling out — qué significa para los planners de search que un tercer actor entra con infraestructura completa en 2026 · qué hacer esta semana 🤖

OpenAI completó esta semana su infraestructura de advertising para ChatGPT: el JavaScript Pixel (browser-side tracking que reporta conversiones cuando un usuario completa una acción en la web del anunciante), la Conversions API (server-side, para un loop de atribución cerrado), CPC bidding, self-serve y las conversion-optimized campaigns —que requieren tener el pixel o la API configurados para poder optimizar hacia outcomes. Los event types disponibles incluyen lead created, order created, page viewed, subscription created y trial started. El sistema ya permite audience targeting por segmento, geographic controls y outcome-based bidding. Cualquier cuenta que tuvo conversiones configuradas antes del 1 de junio recibió acceso anticipado el 5 de junio. La plataforma ya no es "experimental" — es una opción de media con infraestructura de medición comparable a Google o Meta. Para planners: ya no es "ver qué pasa con OpenAI" — es evaluar si ChatGPT Ads tiene escala suficiente para el siguiente cliente con campañas de lead gen o performance.

Por qué importa

OpenAI ya no es el "nuevo actor en search advertising" — es un canal con infraestructura de medición completa que puede competir por presupuesto de performance. La pregunta para los planners ya no es "¿cuándo va a estar listo?" sino "¿qué tipo de cliente y qué objetivo de campaign tiene sentido en ChatGPT Ads vs Google vs Meta?" Para cuentas de lead gen B2B, SaaS, educación o profesionales que buscan intención alta: la audiencia de ChatGPT tiene características diferentes a la de Google Search — y eso merece un test.

🎯 Acción sugerida · Esta semana

(1) Identificar 1-2 clientes con campañas de lead gen o performance en Google que tengan presupuesto para un test incremental: ChatGPT Ads puede correr en paralelo como test sin afectar campañas existentes; (2) Configurar el pixel de OpenAI en el sitio del cliente antes de lanzar — es la base para las conversion-optimized campaigns; (3) Para clientes B2B, SaaS o educación: priorizar el test, porque la audiencia de ChatGPT tiene un perfil de intención alto que puede ser muy eficiente para esas categorías.

OpenAI JavaScript Pixel + Conversions API: la infraestructura de measurement que hace serio a ChatGPT Ads como canal de performance — loop cerrado de atribución — 5 event types disponibles — cómo configurar el tracking antes de lanzar la primera campaña de conversión en ChatGPT · browser-side vs server-side · cuándo usar cada uno 📊

El JavaScript Pixel de OpenAI es la base del measurement de ChatGPT Ads: un tag browser-side que se carga cuando el usuario llega a la página del anunciante y reporta cuando completa una acción definida. Los event types disponibles son: lead created, order created, page viewed, subscription created y trial started. Para un loop de atribución server-side más robusto —menos susceptible al bloqueo de cookies y más preciso para e-commerce— OpenAI también tiene la Conversions API. La cadena completa: anuncio en ChatGPT → clic → pixel o API registra la acción → los datos vuelven a OpenAI → el sistema optimiza hacia el outcome definido. Sin el pixel o la API, las conversion-optimized campaigns no funcionan — el loop de atribución cerrado es el requisito de entrada.

💡

Por qué importa

El pixel y la API de OpenAI convierten a ChatGPT Ads de "plataforma experimental" a "canal de performance medible con atribución". Para el anunciante que quiere correr conversion-optimized campaigns en ChatGPT, el pixel es el paso 0 — y configurarlo ahora (antes de tener budget aprobado) es la diferencia entre lanzar en 24 horas y tener que esperar semanas de setup.
